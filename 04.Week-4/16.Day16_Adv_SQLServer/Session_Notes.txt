 2.  User-Defined Functions
 
 
a.  User-Defined Functions (UDFs) in SQL Server are reusable routines that accept parameters, perform an action (such as a complex calculation or data retrieval), and return the result as a value or a result set. 
 
b. They help in modular programming, reduce network traffic, and can improve performance by caching execution plans.




1.  Scalar Functions: These functions return a single data value of a predefined scalar type (e.g., INT, VARCHAR, FLOAT). The function body is typically enclosed in BEGIN...END blocks and can contain multiple T-SQL statements to achieve a result.

2. Table-Valued Functions (TVFs): These functions return a TABLE data type (a result set of rows and columns).

   Cursors in SQL Server
   ~~~~~~~~~~~~~~~~~~
   
   -->  In SQL Server, a cursor is a database object used to process a result set one row at a time, instead of using the default set-based operations. 
   
   -->  They are useful for tasks that require row-by-row logic, such as administrative tasks or specific conditional updates, but are generally discouraged for performance-critical operations involving large datasets.
   
   
   
Cursor Lifecycle in SQL Server (T-SQL)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
The process of using a cursor involves five main steps: 

1.  Declare: Define the cursor with a name and the SELECT statement that populates its result set.

2. Open: Execute the query and load the result set into the cursor work area.

3. Fetch: Retrieve a single row from the current position into local variables. This process is repeated, typically within a WHILE loop, using @@FETCH_STATUS to check if more rows exist.

4.  Close: Release the current result set and unlock the tables.

5.  Deallocate: Remove the cursor definition and free up all associated system memory resources.
